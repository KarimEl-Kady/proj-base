5/**
6 * Finds App\... references in a PHP file for module:boundaries, using
7 * PHP's own tokenizer instead of a byte-level regex over the raw file.
8 *
9 * Two token kinds count as a reference:
10 *  - T_NAME_QUALIFIED / T_NAME_FULLY_QUALIFIED: a real symbol reference —
11 *    a `use` import, a type hint, `new X`, `X::class`, extends/implements,
12 *    a catch type. This is what a byte-level regex can't distinguish from
13 *    a docblock that merely *mentions* another module's class in passing;
14 *    tokenizing means a comment is never one of these token kinds.
15 *  - T_CONSTANT_ENCAPSED_STRING: a string literal. Kept deliberately,
16 *    because this project's own generators build class names dynamically
17 *    ("App\\Modules\\{$module}\\Models\\..."), and a literal module name
18 *    inside a built string is real signal a pure symbol-reference scan
19 *    would miss. A genuinely interpolated variable segment (the {$module}
20 *    part itself) can't be resolved by any static tool — tokenizer or a
21 *    full AST alike — that limitation is inherent, not specific to this
22 *    implementation.
23 */
